# ヨーロッパ観光・経済分析 - Microsoft Fabric ポートフォリオ

🇬🇧 English version → [README.md](README.md)

## 概要

Microsoft Fabric Lakehouseを使用して、観光・気候・経済という複数のデータソースを統合した分析基盤を構築したデータエンジニアリングプロジェクトです。

### 使用技術

Microsoft Fabric Lakehouse | PySpark | Spark SQL | Delta Lake | Open-Meteo API | Power BI

## 目的

Microsoft Fabric Lakehouseを活用したマルチソースデータ統合基盤の設計・構築。APIベースのデータ取得、CSVデータ処理、データ変換、レポーティングまでを一貫して実装しました。

## データソース

- **Kaggle**：World Tourism Economy Data（観光客数・GDP・インフレ率／国別）
  - 対象期間：2000年〜2020年（観光客数）、2000年〜2023年（インフレ率）
- **Open-Meteo Archive API**：ヨーロッパ12都市の過去の気温データ
  - 対象期間：2000年〜2023年

> **補足：** 観光客数データ（2021〜2023年）は、新型コロナウイルス感染症の影響によるデータ収集の中断により、元データセットに含まれていません。

## アーキテクチャ

メダリオンアーキテクチャ（Bronze / Silver / Gold）

### Bronze

- Kaggleの生CSVファイルをDeltaテーブルに格納（`bronze_tourism`）
- Open-Meteo Archive APIから都市グループごとに気温データを取得し、Deltaテーブルに格納（`bronze_temperature`）

### Silver

- 国コードによりヨーロッパ12カ国にフィルタリング
- 分析用途に合わせてカラムを選定・リネーム（`silver_tourism`）
- 気温データは変換不要のため、Bronzeのデータをそのまま使用

### Gold

- 観光・気温・インフレデータを `country_code` と `year` でJOIN
- 単一の分析用テーブルとして格納（`gold_european_analysis`）

## 技術的なポイント

- Microsoft Fabric Lakehouseを用いて、マルチソースデータ取得からPower BIレポートまで一貫したデータ基盤を設計・構築
- REST API（Open-Meteo）とCSV（Kaggle）という異種データソースを統合し、単一のDelta Lakeテーブルとして整備
- PySparkを用いてOpen-Meteo Archive APIからヨーロッパ12都市の過去の気温データ（2000〜2023年）を取得
- Delta Lakeテーブルによるメダリオンアーキテクチャ（Bronze / Silver / Gold）を実装
- PySparkおよびSpark SQLによるデータ変換・JOIN処理を開発
- Power BIレポーティングに最適化した単一テーブルのGoldレイヤーを設計

## 分析から得られた知見

- 2020年は新型コロナウイルスの影響で観光客数が大きく落ち込み、2021〜2023年は元データセットにデータが存在しません。
- クロアチアはフランス・スペイン・イタリアに次ぐ観光客数4位を記録しており、経済規模を考えると意外な結果でした。
- 平均気温と観光客数の間に強い相関は見られませんでした。観光客数1位のフランスも、気候自体は穏やかな部類です。
- 2010年は全都市で気温が大きく低下しており、これは2009〜2010年にかけての北大西洋振動（NAO）の負位相による、ヨーロッパ全体の記録的な寒波と一致しています。
- ハンガリーは2021年以降インフレ率が急上昇し、2022〜2023年には対象12カ国の中で最も高いインフレ率を記録しました。
